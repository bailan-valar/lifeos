# 待办状态管理功能

## 功能概述

待办状态管理功能允许用户为待办事项设置不同的状态，例如：待办、进行中、已暂停、已完成等，从而更好地跟踪和管理任务的进度。

## 功能特性

### 1. 状态管理
- **自定义状态**：用户可以创建自定义的待办状态
- **状态图标**：每个状态可以设置独特的图标和颜色
- **默认状态**：可以设置一个默认状态，用于新建待办时的初始状态
- **状态排序**：状态可以按照用户的需要进行排序
- **状态描述**：每个状态可以添加详细的描述信息

### 2. 默认状态
系统预置了以下默认状态：
- **待办**：待处理的任务（蓝色，圆形图标）
- **进行中**：正在进行的任务（橙色，时钟图标）
- **已暂停**：暂时停止的任务（红色，暂停图标）
- **已完成**：已完成的任务（绿色，完成图标）

### 3. 状态显示
- 在待办列表中以彩色标签形式显示状态
- 支持鼠标悬停查看状态描述
- 状态标签与完成状态可以并存

### 4. 状态管理界面
- 直观的状态管理对话框
- 支持创建、编辑、删除状态
- 可以设置默认状态
- 支持重置为默认状态

## 使用方法

### 基本使用

1. **打开状态管理**
   - 在待办视图中点击右上角的设置图标（齿轮图标）
   - 打开状态管理对话框

2. **创建新状态**
   - 点击"新建状态"按钮
   - 填写状态名称、描述
   - 选择图标和颜色
   - 选择是否设为默认状态
   - 点击"创建"按钮

3. **编辑状态**
   - 在状态列表中找到要编辑的状态
   - 点击编辑按钮（笔图标）
   - 修改状态信息
   - 点击"保存"按钮

4. **删除状态**
   - 在状态列表中找到要删除的状态
   - 点击删除按钮（垃圾桶图标）
   - 确认删除操作
   - 注意：默认状态不能删除，正在被使用的状态不能删除

5. **设置默认状态**
   - 在状态列表中找到要设为默认的状态
   - 点击星标图标按钮
   - 该状态将被设为默认状态

### 在待办项中使用状态

1. **新建待办时**
   - 新建的待办会自动使用当前默认状态
   - 可以在待办编辑对话框中修改状态

2. **修改待办状态**
   - 在待办编辑对话框中选择新的状态
   - 状态会立即更新显示

## 技术实现

### 数据结构

```typescript
interface TodoStatus {
  id: string              // 状态唯一标识
  name: string            // 状态名称
  icon: string            // 状态图标
  color: string           // 状态颜色
  description?: string    // 状态描述
  order: number           // 排序顺序
  isDefault: boolean      // 是否为默认状态
  createdAt: string       // 创建时间
  updatedAt: string       // 更新时间
}
```

### 核心组件

1. **useTodoStatus**：状态管理的composable
   - 提供状态的增删改查功能
   - 管理默认状态
   - 处理状态初始化

2. **TodoStatusBadge**：状态显示组件
   - 在待办列表中显示状态标签
   - 支持自定义颜色和图标
   - 响应式设计

3. **TodoStatusSelector**：状态选择组件
   - 在编辑对话框中选择状态
   - 可视化的状态选项
   - 支持实时预览

4. **TodoStatusManageDialog**：状态管理对话框
   - 综合的状态管理界面
   - 支持所有状态操作
   - 直观的用户体验

5. **TodoStatusEditDialog**：状态编辑对话框
   - 创建和编辑状态
   - 图标和颜色选择器
   - 实时预览功能

## 集成到待办视图

待办视图现在集成了状态管理功能：

1. **头部操作栏**
   - 添加了状态管理按钮
   - 可以快速访问状态管理界面

2. **待办列表显示**
   - 每个待办项显示其当前状态
   - 状态以彩色标签形式显示
   - 支持鼠标悬停查看详细信息

3. **状态初始化**
   - 新建待办时自动使用默认状态
   - 支持状态的自定义和扩展

## 数据持久化

状态数据使用PouchDB进行持久化存储：
- 集合名称：`todoStatuses`
- 支持实时同步
- 支持离线使用
- 自动数据备份
