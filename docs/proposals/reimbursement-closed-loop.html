报销功能闭环优化

垫付 → 归集 → 回款 → 闭环
多笔支出，一笔报销，全程可追踪。

💡 报销 ≠ 退款
维度 退款 Refund 报销 Reimbursement
场景 买的东西退了，商家退钱 替公司垫付，公司后付款给我
示例 网购退货 → 商家退款 出差打车 ¥30 → 公司报销 ¥30
关系 1 : 1（一笔支出 ↔ 一笔回款） N : 1（多笔支出 ↔ 一笔回款）
现有支持 已实现 未实现

字段关系说明

isReimbursable reimbursementId 含义 badge 显示
true 空 可报销，还没创建报销单 [可报销]
true 有值 正常报销流程中 [待报销]
false/空 空 普通支出，无报销需求 —
false 有值 异常：不可报销但加入了报销单 [⚠️ 异常]
📊 预算排除规则
核心需求
报销的支出不计入项目预算和分类预算。退款同样不计入。
已由公司报销或商家退款的支出，不应消耗预算额度。

排除规则

账单状态 是否计入预算 说明
普通支出 计入 正常消耗预算
父账单（hasChildren） 排除 只计子账单，避免重复
退款账单（isRefund=true） 排除 已退回，不消耗预算
已退款的原账单 计入 但减退款额 原支出仍消耗，退款部分冲回
待报销支出（报销单未 paid） 计入 还没拿到钱，仍消耗预算
已报销支出（报销单 = paid） 排除 公司已付款，不消耗我的预算
部分报销支出 计入差额 只计自付部分（支出 - 已报销）

计算公式

实际预算消耗
实际预算消耗 =
+ 全部子支出（排除 hasChildren）
- 退款总额（isRefund=true 的收入）
- 已报销总额（报销单=paid 的支出）
