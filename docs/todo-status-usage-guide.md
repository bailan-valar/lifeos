# 待办状态管理功能使用指南

## 快速开始

### 1. 访问功能
- 打开待办视图模块
- 点击右上角的齿轮图标打开状态管理

### 2. 首次使用
系统会自动创建4个默认状态：
- 🔵 **待办** - 待处理的任务
- 🟠 **进行中** - 正在进行的任务  
- 🔴 **已暂停** - 暂时停止的任务
- 🟢 **已完成** - 已完成的任务

## 基本操作

### 创建自定义状态
1. 打开状态管理对话框
2. 点击"新建状态"按钮
3. 填写状态信息：
   - **名称**：状态的显示名称（如："审核中"、"待确认"）
   - **描述**：状态的详细说明（可选）
   - **图标**：选择合适的图标
   - **颜色**：选择醒目的颜色
   - **默认状态**：勾选后新建待办时自动使用此状态
4. 点击"创建"完成

### 编辑现有状态
1. 在状态列表中找到要修改的状态
2. 点击编辑按钮（笔图标）
3. 修改状态信息
4. 点击"保存"完成

### 删除状态
1. 在状态列表中找到要删除的状态
2. 点击删除按钮（垃圾桶图标）
3. 确认删除操作

**注意**：
- 默认状态不能删除
- 正在被待办项使用的状态不能删除

### 设置默认状态
1. 在状态列表中找到要设为默认的状态
2. 点击星标图标按钮
3. 该状态将成为新建待办的默认状态

## 在待办项中使用状态

### 新建待办
- 新建的待办项会自动使用当前默认状态
- 可以在待办编辑中修改状态

### 修改待办状态
1. 点击待办项的编辑按钮
2. 在状态选择器中选择新的状态
3. 保存修改

### 查看待办状态
- 状态会在待办列表中以彩色标签显示
- 鼠标悬停可以查看状态的详细描述

## 状态管理最佳实践

### 状态设计原则
1. **简洁明了**：状态名称要清晰易懂
2. **颜色区分**：使用不同颜色区分不同类型的状态
3. **适度数量**：建议3-8个状态，避免过多造成混乱
4. **逻辑清晰**：状态之间应该有明确的流转关系

### 推荐的状态设置
**简单模式**（3-4个状态）：
- 待办 → 进行中 → 已完成

**标准模式**（5-6个状态）：
- 待办 → 进行中 → 待审核 → 已完成 → 已取消

**复杂模式**（7-8个状态）：
- 待办 → 进行中 → 待审核 → 待确认 → 已完成 → 已暂停 → 已取消

### 颜色选择建议
- 🔵 **蓝色系**：表示初始、待处理状态
- 🟠 **橙色系**：表示进行中、需要关注的状态
- 🔴 **红色系**：表示紧急、停止、错误状态
- 🟢 **绿色系**：表示完成、成功状态
- 🟣 **紫色系**：表示特殊、高级状态

### 状态流转规则
- 定义明确的状态流转路径
- 避免状态之间的随意跳转
- 记录状态变更的历史

## 常见问题解答

### Q: 如何恢复默认状态设置？
A: 在状态管理对话框中点击"重置为默认"按钮，将恢复系统的4个默认状态。

### Q: 可以修改默认状态的图标和颜色吗？
A: 可以，编辑默认状态修改其图标和颜色，不影响其功能。

### Q: 删除状态后，使用该状态的待办项会怎样？
A: 系统不允许删除正在被使用的状态，需要先修改相关待办项的状态。

### Q: 如何在待办列表中按状态筛选？
A: 当前版本支持状态显示，筛选功能将在后续版本中添加。

### Q: 状态数据会同步到其他设备吗？
A: 是的，状态数据通过PouchDB进行实时同步，确保多设备数据一致。
