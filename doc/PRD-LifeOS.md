## 1. 产品概述

### 1.1 产品愿景

LifeOS 不是另一个生产力工具，它是用户的**生活操作系统**。通过场景感知、智能捕获、无缝集成，帮助用户从信息焦虑中解放，做正确的事，而不是更多的事。

### 1.2 核心价值主张

- **从15分钟到5分钟**：将每日任务管理时间从15分钟缩减到5分钟
- **从容应对临时任务**：20%的临时任务不再打断工作流
- **统一知识库**：所有信息在一个系统，智能搜索，秒速找到
- **全流程可视化**：项目进度透明，每个环节都在掌控中

### 1.3 产品定位

**不是：**
- 单纯的任务管理器
- 笔记软件
- 财务工具

**是：**
- 生活操作系统
- 第二大脑
- 志向与日常的桥梁

## 2. 目标用户

### 2.1 主要用户画像

**核心用户：知识工作者（产品经理、程序员、设计师、创业者）**

- 年龄：25-40岁
- 工作经验：5年以上
- 痛点：
  - 信息过载，多个渠道（微信、邮件、口头）的任务难以统一管理
  - 每天花15分钟排序任务，仍然感到焦虑
  - 知识分散在多个工具，关键时刻找不到
  - 项目进度不透明，容易遗漏重要节点

### 2.2 使用场景

- **日常管理**：每天早上花5分钟确认当日重点
- **临时任务处理**：微信/邮件/口头想法，快速捕获并分类
- **项目追踪**：查看所有项目的生命周期状态
- **财务洞察**：每周/每月查看收支趋势和消费分析
- **目标对齐**：每周回顾OKR进度，每日完成习惯追踪

## 3. 核心功能需求

### 3.1 场景感知 (Scene Awareness)

**优先级：** P0（MVP必须）

**功能描述：**
根据时间、用户习惯自动切换场景，聚焦当下最重要的任务。

**用户故事：**
- 作为用户，我需要在工作时间（8:30-11:30, 14:00-17:30）自动进入"工作场景"，只看到工作相关的任务和项目
- 作为用户，我需要在周一晚上19:30-20:30自动进入"家庭场景"，提醒我家庭待办
- 作为用户，我需要在周三晚上19:00-20:30自动进入"财务场景"，快速查看财务报告

**功能需求：**
1. **场景定义**
   - [ ] 支持自定义场景（工作、生活、娱乐、健康、学习、副业）
   - [ ] 每个场景关联特定的任务、项目、目标
   - [ ] 每个场景可设置图标、颜色

2. **时间触发**
   - [ ] 支持固定时间触发（如工作日8:30自动切换到工作场景）
   - [ ] 支持周期性触发（如每周一19:30自动切换到家庭场景）
   - [ ] 支持手动切换场景

3. **场景视图**
   - [ ] 切换场景后，只显示该场景下的任务、项目、目标
   - [ ] 顶部显示当前场景名称和图标
   - [ ] 显示场景切换历史

4. **场景提醒**
   - [ ] 场景切换时，显示该场景下今日待办
   - [ ] 可配置是否在场景切换时发送通知

**验收标准：**
- 用户定义了工作场景（8:30-11:30），系统在8:30自动切换
- 工作场景下只显示工作相关任务
- 用户可手动切换到其他场景

**负责人：** [待填写]
**指标：**
- 场景切换成功率 > 95%
- 用户手动切换频率 < 10%（说明自动切换准确）
**时间线：** Sprint 1 (Week 1-2)

### 3.2 项目引擎 (Project Engine)

**优先级：** P0（MVP必须）

**功能描述：**
每个项目有专属仪表盘，支持自定义字段、生命周期可视化、进度追踪。

**用户故事：**
- 作为用户，我需要为项目添加自定义字段（如"优先级"、"截止日期"、"负责人"）
- 作为用户，我需要看到项目的生命周期状态（规划→执行→测试→上线）
- 作为用户，我需要一眼看到所有项目的进度和关键信息

**功能需求：**
1. **项目创建与编辑**
   - [ ] 创建项目，设置项目名称、描述
   - [ ] 添加自定义字段（文本、数字、日期、选项）
   - [ ] 设置项目生命周期阶段（规划、执行、测试、上线）
   - [ ] 设置项目负责人、截止日期、关键指标

2. **项目列表视图**
   - [ ] 看板视图：按生命周期阶段分组显示项目
   - [ ] 列表视图：表格形式显示所有项目
   - [ ] 支持按场景、状态、负责人筛选
   - [ ] 支持搜索项目

3. **项目详情页**
   - [ ] 显示项目基本信息（名称、描述、负责人、截止日期）
   - [ ] 显示项目自定义字段
   - [ ] 显示项目生命周期进度
   - [ ] 关联任务列表
   - [ ] 项目时间线（里程碑）

4. **项目权限**
   - [ ] 项目可设置为公开或私密
   - [ ] 私密项目仅项目负责人可见

**验收标准：**
- 用户可以创建项目，添加3种以上自定义字段
- 项目看板按生命周期分组显示
- 项目详情页展示完整信息

**负责人：** [待填写]
**指标：**
- 项目创建成功率 > 98%
- 项目字段配置完成率 > 80%
**时间线：** Sprint 2 (Week 3-4)

### 3.3 任务中心 (Task Center)

**优先级：** P0（MVP必须）

**功能描述：**
智能捕获临时任务（微信、邮件、口头想法），自动分类，支持每日优先级排序。

**用户故事：**
- 作为用户，我需要将微信消息快速转换为任务
- 作为用户，我需要AI帮助我提取邮件中的关键信息并创建任务
- 作为用户，我需要每天花5分钟确认今日最重要的事
- 作为用户，我需要任务自动关联到场景或项目

**功能需求：**
1. **任务创建**
   - [ ] 手动创建任务
   - [ ] 快速创建（快捷键、浮动按钮）
   - [ ] 任务必填字段：标题、截止日期、优先级
   - [ ] 任务可选字段：描述、场景、项目、标签

2. **智能捕获**
   - [ ] 微信消息转任务（MVP后）
   - [ ] 邮件转任务（MVP后）
   - [ ] 语音转文字创建任务（MVP后）
   - [ ] AI提取关键信息（标题、截止日期、优先级）

3. **任务列表**
   - [ ] 今日任务视图
   - [ ] 按优先级排序（高、中、低）
   - [ ] 按场景筛选
   - [ ] 按项目筛选
   - [ ] 已完成/未完成切换

4. **每日优先级排序**
   - [ ] 每日早上8:30自动推送"今日重点"
   - [ ] AI推荐今日最重要的3-5个任务
   - [ ] 用户可手动调整任务优先级

5. **任务详情**
   - [ ] 显示任务所有字段
   - [ ] 任务评论
   - [ ] 任务子任务
   - [ ] 任务附件

6. **任务状态**
   - [ ] 待办
   - [ ] 进行中
   - [ ] 已完成
   - [ ] 已取消

**验收标准：**
- 用户可以手动创建任务
- 今日任务视图按优先级排序
- 任务可以关联到场景和项目
- AI推荐今日任务（MVP后）

**负责人：** [待填写]
**指标：**
- 任务创建成功率 > 99%
- 每日任务完成率 > 60%
- 用户平均每日任务管理时间 < 5分钟
**时间线：** Sprint 1 (Week 1-2)

### 3.4 财务掌控 (Finance Control)

**优先级：** P1（MVP+）

**功能描述：**
实时数据，智能洞察，可视化收支趋势，消费分析，财务目标追踪。

**用户故事：**
- 作为用户，我需要看到每月的收支趋势
- 作为用户，我需要AI识别异常消费并给出建议
- 作为用户，我需要追踪我的财务目标（买房、旅行、投资）进度

**功能需求：**
1. **收支记录**
   - [ ] 手动添加收支记录
   - [ ] 收支分类（餐饮、交通、娱乐、医疗、教育等）
   - [ ] 支持导入银行账单（CSV）
   - [ ] 收支备注

2. **数据可视化**
   - [ ] 月度收支趋势图
   - [ ] 分类支出饼图
   - [ ] 年度财务概览
   - [ ] 对比分析（本月vs上月）

3. **消费分析**
   - [ ] AI识别异常消费（超过平均水平的支出）
   - [ ] 消费建议（如"本月餐饮支出比上月高20%"）
   - [ ] 消费预警（预算超支提醒）

4. **财务目标**
   - [ ] 创建财务目标（如"存10万买房"）
   - [ ] 设置目标金额、截止日期
   - [ ] 追踪目标进度
   - [ ] 目标完成提醒

5. **智能提醒**
   - [ ] 信用卡还款提醒
   - [ ] 定期存款提醒
   - [ ] 预算超支提醒

**验收标准：**
- 用户可以手动添加收支记录
- 月度收支趋势图正常显示
- AI能识别异常消费
- 财务目标进度正确计算

**负责人：** [待填写]
**指标：**
- 收支记录准确率 > 99%
- 异常消费识别准确率 > 80%

## 5. 非功能性需求

### 5.1 性能要求

- 应用启动时间 < 3秒
- 页面切换时间 < 500ms
- 搜索响应时间 < 1秒
- 支持10,000+任务流畅运行

### 5.2 可用性要求

- 系统可用性 > 99.5%
- 数据丢失率 < 0.01%

### 5.3 安全性要求

- 用户数据本地存储，不上传云端（默认）
- 数据加密存储（AES-256）
- API Token鉴权
- 密码哈希存储（bcrypt）

### 5.4 兼容性要求

- **客户端平台**：Windows、macOS、Linux
- **移动端**：iOS、Android（MVP后）
- **浏览器**：Chrome、Firefox、Safari、Edge（Web版）

### 5.5 可访问性要求

- 支持键盘快捷键
- 支持屏幕阅读器
- 高对比度模式
